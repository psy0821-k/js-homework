## 서론

이번 과제를 하면서 어떻게 하면 코드량을 줄일 수 있을까 고민을 많이한 것 같습니다. 그리고 지금 배운 범위에 근접한 범위 내에서 어떻게 작성해야 좀 더 유지보수성이 좋은 혹은 성능이 괜찮은 코드가 나올까 라는 생각을 갖고 과제를 한것 같습니다

## 과제 수행 기록

1. 파일 분리하기 
파일을 처음에 어떻게 분리를 해야할까? 고민을 했습니다. 기능별로 하나하나 나누기엔 아직 파일을 import나 export하는 방법을 아직 배우지 않았기 때문에 과제를 크게 변수선언 과 함수선언 두가지로 보고 나누었습니다.
2. 명시적으로 작성하기
- 불리언으로 나타나는 변수명은 is를 붙였습니다
- 최댓값인 경우 나타내는 그 최댓값이 이하,이상 을 기준으로 변수 명에 in을 섞었습니다. 초과 혹은 미만이면 Limit를 추가
- 세미콜론 붙이기 ( 수업시간에는 안붙이는 방법으로 하지만 명시적으로 코드를 작성해보는것도 중요하다고 생각했습니다 )
- 임시변수 자제하기 ( 아직 이 부분에 대한 이해도가 부족하지만 최대한 노력해보았습니다. )
- 조건문 (else if / else 자제하기) if문을 여러번 사용하여 명시성을 높였습니다. 
( 과하다면 switch case문이 적합하지만 이번과제는 과하지 않다고 생각했습니다 ) 
- 객체 최대한 활용하기 
( 객체를 마지막으로 배운만큼 활용을 많이 해보자고 생각했습니다. 하지만 지금 이렇게 하는것이 좀 더 좋은 코드를 만드는 방법인가? 아니면 단순하게 가는것이 더 좋지 않을까? 라는 생각이 있습니다 )

#### 1번 랭크 별 환영 메세지
1번 문제의 경우 변수 과제에 랭크 점수가 있다면 그 점수를 활용해서 코드를 작성해 보면 좋을 것 같았습니다
기본값을 바로 반환하되 만약 vip라면 문제의 문구를 반환했습니다 
여기서 추가적으로 저는 객체를 선언해서 작성했기 때문에 
회원이 아닐경우 이름이 undefined님 즐거운 쇼핑되세요가 나올것을 대비해 if문을 추가적으로 작성해 주었습니다

그 이후 각 객체를 콘솔로그를 이용하여 출력 후 값을 확인했습니다

#### 2번 배송비
기본 배송비를 처음에 지정해 주었습니다. 이 값은 바뀌면 안된다고 생각했습니다.
그리고 배송비를 마지막에 도출 해 줄 변수를 선언해 계산해 주었습니다
첫번째 조건인 50000원 이상 그리고 0원 미만은 기본 배송비를 0원으로 계산이 되었습니다
조건문에 논리속성인 or을 사용했고 0원으로 재할당 해주었습니다. 바로 반환하지 않은 이유는 그 가격에 조건을 맞추어도 도서, 제주 지역은 배송비 추가 대상이기 때문입니다.

각 지역은 서울과 제주 지역으로 그리고 금액은 조건을 확인할 수 있을 정도만 실행해 보았습니다.

#### 3번 비밀번호 유효성 검사

비밀번호의 경우 .length 속성을 통해 길이를 구한 후 참 거짓을 반환했습니다. 
if문 으로 작성 후 바로 반환하는 것이 가능해 보였습니다.

#### 4번 포인트 적립 과제
처음엔 if문으로 랭크를 하나하나 나누었습니다.
그렇다보니 제 눈에는 중복된 부분이 많아보였습니다 이를 해결하기위해 객체를 사용해보자 라는 생각이 떠올랐습니다. 
처음엔 객체안에 키 값으로 랭크로 나누었습니다 그리고 값으로 가격과 해당 랭크가 무슨랭크인지 값으로 넣었습니다. 반환 값을 쓰고 생각해 보니 랭크가 어떤랭크인지 표기를 할 필요도 없고 더 간단한 방법이 있었습니다

NaN에러 제가 등급을 소문자로 받고 키 값은 대문자로 바꿔주어서 다음과 같은 에러가 발생했습니다. 
이를 해결하기 위해 toUpperCase() 메서드를 사용해 간단하게 해결했습니다

그래서 코드 수정을 했습니다.
