# 1월 3주차 과제기록 과 회고

## 디자인 분석하기

디자인 분석을 함에 있어서 마크업 구조는 금방 잡혔는데 세부 디자인의 경우는 이리저리 피그마와 css파일을 자주 왔다갔다 하면서
작성을 작성을 했습니다. 다 끝마친 후엔 분석한 디자인을 css 파일에 주석으로 적었다면 좀 더 효율이 올라갔을것 같다는 생각이 들었습니다.

## html 마크업

1. 과제를 마크업 함에 있어서 먼저 폴더 구조를 잡았습니다.
   막상 혼자 폴더구조를 잡으려니 이 파일을 이렇게 나누면 될 지 파일 명은 어떤게 좋을지 고민을 하게 되는것 같습니다. 그러다보면 이게 좋은 폴더구조를 가지고 있는지 의심을 품게 되었습니다. 이번 과제때는 선생님의 폴더구조를 참고하여 작성을 했음에도 많이 부족함이 느껴집니다.
2. 일단 마음가는대로 aria는 생각하지 않고 시멘틱한 구조에 맞추어
   마크업을 진행해 보았습니다. 그 후 aria 속성을 추가하고 css를 작성하면서 부족한 클래스는 채워넣고 필요한 클래스는 사용했습니다.클래스를 작성하면서 일관성 있게 작성해야하는데 어디서는 케밥케이스 어디서는 카멜케이스를 쓰고 있는 저 자신의 모습을 보았습니다. 이 클래스는 css작성당시 문제를 안 일으켰지만 자바스크립트 작성시 class명이 문제를 일으키면서 명시적이고 일관성 있는 변수명을 작성하는것이 중요하다는것을 한번더 느꼈습니다
3. validata검사 생각보다 에러가 많이났습니다. 하지만 이를 해석하고 수정을 함에 있어서 빠르진 않아도 천천히 해결 할 수 있음에 성장하고 있음을 느꼈습니다. 하지만 오류를 내지않고도 코드를 작성할 수 있는 사람이 되고싶다는 생각도 드네요

## css 작성

### 문제와 해결방안

처음엔 저 배경안에 캐릭터 이미지가 밖으로 나오면서 일정부위만 가려지게 어떻게 해야할 지 고민이 많았습니다. 하지만 이리저리 하다보니 우리 눈에 안보이는 또하나의 영역을 만들면 되겠다 라는 결론이 나왔습니다. 단순 검색이 아닌 내 생각에서 나온 결과가 css를 작성함에 있어서 좀 더 이해도 있는 코드를 작성 할 수 있었다고 생각합니다.

### 라인하이트

코드를 다 작성하고 나니 디자인 상 전체높이와 제 전체 높이가 다르더군요.. 이를 해결하기위해 line-hight를 떠올렸으나 막상 맞는지 잘 모르겠다 라는 생각이 더 컸었습니다. 이 문제는 피그마에서 금방 해결 되었습니다. 친절하게도 줄 높이가 몇인지 css어떻게 적용하면 될지 바로 알 수있게끔 표시가 되어있었습니다.
문제에 대한 해결방안은 어쩌면 디자인을 분석할 때 나타나기도 하는것 같습니다.

### css검사

검사를 하니 네스팅을 사용하면 경고가 잔뜩 떴습니다 그 경우엔 저는 무시했습니다. 왜냐하면 슬비쌤과 네스팅 기법을 사용하여 css를 작성했기 때문인데요 경고는 경고일 뿐 에러가 아니기 때문에 문제 없다고 생각했습니다.
버튼에 hover시 백그라운드 색상 에러가 났습니다. 이 경우에도 무시했습니다. 접근성도 중요하지만 이 경우엔 디자이너의 의도에
맞춰서 색상을 적용해야 된다는 생각이 더 크기 때문입니다.
layer를 사용하니 빨간 내용 즉 에러나 경고가 잔뜩 떴습니다. 어떤 상황에서 layer을 사용하는지는 알고있으나 막상 검사를 하고나니 겁이나서 다 삭제한 것 같습니다. 이 경우엔 제 이해도가 아직 많이 미흡하다고 생각하기 때문에 한번 알아봐야겠다는 생각이 들기도 합니다
