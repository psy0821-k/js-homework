# 1월 25일 회고

## 이번 과제를 하면서
과제를 이번에 하면서 결국 ai에 손을 대버린 것에 대해 아쉬움이 남습니다. 
이번엔 회고를 남은시간이 얼마 남지않아 급하게 써보려고 합니다

ai사용 한 case
- innerHtml
수업시간에 언급한 innerHtml을 사용하여 코드를 입력해 주었습니다 코드를 보시면 아직 배우지 않은 삼항연산자와 변수를 집어넣을 수 있는 ${}을 넣어야 스타일을 넣을 수 있었습니다. 단순 setProperty를 사용하고 싶었으나 innerHtml을 사용했을 땐 전역변수 혹은 변수에 nodeElementChild? 이름도 기억이 잘 나지 않습니다. 배운 기억은 슬며시 나는것 같았으나 모르는 코드를 그냥 ai가 알려주는대로 복붙할 수 없다는 생각에 이전에 알고있었던 것만 사용하며 코드를 이해하려 했습니다 다만 이 코드 어떻게 작동하는 지 순서에 좀더 생각해 봐야할 것같습니다.

- 오브젝트
객체를 생성해서 데이터를 넣었습니다. 이전에 한번 본적이 있는 mongoDB가 생각 났습니다. 사용하는 방법은 많이 다르겠지만 데이터가 저장되는 방식은 같은 객체형태였던것 같습니다. 1차로 객체를 생성한 후에 너무 지저분하다는 생각이 듦과 동시에 나는 백엔드가 아니니까 상관없지 않을까? 라고 생각했습니다. 하지만 제가 나중에 코드를 작성할 때도 불편해 지겠다 생각해서 2차로 데이터형태를 수정했습니다. 사용자가 있고 그 사용자 안에 계좌나 카드 등 여러 데이터를 넣는 방식으로 넣었습니다. 하지만 너무 귀찮은 나머지 ai를 통해 너가 이 형태에 로 바꾼 후 코드를 나에게 보여봐! 했을땐 거의 90%는 편리하게 만들수 있었습니다. 다만 다 넣어주지 않고 일부는 제 의도와는 다르게 분리해서 보여주었습니다. 제가 수정해서 넣었습니다, 하지만 이런 간단 반복작업은 ai에게 시켜도 되겠다 생각을 했습니다.

- 돈을 읽어주는 aria-lable 만들기
저는 돈을 읽어줄 떄 스크린 리더가 일이삼사오육칠원 이렇게 읽어버리지 않을까?라는 생각에서 시작을 했습니다

ai 사용하기전 전 먼저 작성을 하는 연습을 했습니다. 숫자를 텍스트로 만들 때 어떻게 추가해줘야할지 감이 잘 오지 않았습니다. 그래서 ai에게 힌트를 얻고자 사용을 했습니다. 다만 ai는 1=일로 2는 이로 하나하나 다 바꾸어 주었습니다. 그런데 저는 이렇게까지 바꿔야 하나?라는 생각이 들어 그 참고사항을 기반으로 제 방식 대로 다시 작성해 보았습니다. 제 코드가 좋다고는 생각 안합니다.
```
let calculaterMoney = defaultMoney - currentMoney * 10000;
```
위 코드만는 만원 미만의 돈을 읽어주기위해 만든 코드입니다 %10000을 사용했으면 더 간단했을것 같습니다.
하지만 ai에게 약간의 힌트를 얻었을지라도 스스로 코드를 작성하고 완성시킨것에 지금은 의의를 두고자 합니다
앞으로 더 나은 개발자가 될 수 있도록 배우는 단계라 과제할 때 만큼은 ai를 사용하지말아야지 라는 마음을 갖고 노력해야겠습니다

## 마지막으로

마음이 급하다보니 제가 작성한 코드에 대한 설명을 다 적지는 못했습니다. 그 중 이렇게 ai를 활용했습니다. 이런 어려움이 있었습니다.라는 핵심적인 이야기를 주저리주저리 적은것 같습니다. 하지만 마음이 급하니 ai가 주는 코드를 이해를 했으면 대충 복사 붙여넣기를 하게되고 마음에 들지도 않고 사용하기 싫은 코드를 사용한것 같습니다. 특히 수기 이미지를 넣은 페이지가 그 예시입니다 style에 이미지도 최적화 안되어 있고 너무너무 불편한 페이지입니다. 지금은 마무리했다는 후련함과 만족스럽지 못한 찝찝함이 공존하는 상태에서 제출시간이 지났기에 일단 제출하고자 합니다.
