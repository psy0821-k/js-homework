# 서론

이번 과제를 하기 전 저번에 한 과제 때를 생각해보니 지금 내가 하는 마크업이 맞는지도 아리아 속성을 제대로 사용하고 있는지도 모르는데 이대로 괜찮은가? 라는 생각이 들었습니다. 과제를 마치고 나서 시각적인건 고치겠지만 내부 시스템적인것까지 고치려니 힘이 너무 빠져 다음것을 진행 할 힘이 나지 않았습니다. 이를 보완하기위해 1차적으로 내가 코드를 작성하고 다음 선생님의 코드를 참고하여 수정하면 어떨까 라는 생각이 떠 올랐습니다.

# 본론

## 진행방법

1. 1차적으로 마크업 및 css코드 작성 진행
2. 선생님의 코드를 보고 수정
3. 선생님의 코드를 보고 중복코드 제거

## 코드 작성 후 선생님의 코드를 보며 개선한 사항

1. h1태그에 들어갈 내용
   (요금제 선택 페이지) 명사로 들어가지 않고 문장으로 제목을 명시해도 괜찮다. 페이지라는 말 보다 서비스라는말이 듣기에 좀더 유연할 수있겠다라는 생각도 듭니다
   효과 : 좀 더 사용자가 이해하기 명확하고 부드러운것 같이 느꼈습니다.

2. section 안의 article
   article로 묵는것 까진 좋았으나 무분별하게 리스트의 형태로 묶은 것 같습니다. 이 전 수업시간에 코드를 볼 때 article안에 section태그가 들어간 것을 본적이 있습니다. 현재 코드를 보니 이게 정해진게 아니라는 것을 알게 되었습니다.
3. 한국어를 기준으로 설정한 파일중 태그에 lang속성을 통해 영어임을 알려줄 수 있다.
4. 보이는 것에 속지 말자
   저는 카드영역이 클릭이 되어야 하기 때문에 이를 input의 타입 radio를 생각해 카드영역에 role을 추가하여 코드를 작성했습니다. 카드영역은 단순 보이는 것일 뿐 버튼 역할은 버튼에게 주어야 한다는 것을 인지해야 할 필요가 있을 것 같습니다
5. section으로 묶을 시 섹션 내에 있는 제목과 연결해 줄 필요가 있다.
6. 현재 헤더나 푸터가없지만 메인태그안에 코드를 작성했다면 좀 더 시멘틱 적이었을것 같습니다
